def print_stats(classifier, prepend_text, X_te, y_te):
    y_te_pred = classifier.predict(X_te)
    print(prepend_text, 'confusion matrix:\n', sklm.confusion_matrix(y_te, y_te_pred))
    print(prepend_text, 'precision:', sklm.precision_score(y_te, y_te_pred))
    print(prepend_text, 'recall:', sklm.recall_score(y_te, y_te_pred))
    print(prepend_text, 'f1:', sklm.f1_score(y_te, y_te_pred))

def do_pca(X, y, label):
    pca = skld.PCA(n_components = 2)
    pca_points = pca.fit_transform(X)

5. (20 points) Explain the main differences between K-means, K-medoids, DBScan and Gaussian Mixture Models (examples are welcome).

K-means is probably the simplest clustering algorithm to understand. It starts with randomly-selected k centroids. Then, for each iteration, it assigns each data point to the closest centroid, then recomputes the centroid's new location by finding the mean of the points assigned to it. This can lead to problems, however, if the number of clusters the user specifies is not the same as the actual number of clusters in the dataset. For example, if you enter 2 clusters for data with 3 clusters, it's possible that the centroid ends up halfway between two of the clusters.

To fix this problem, K-medoids is introduced to ensure that the location for a centroid "makes sense." Rather than assigning the centroid's new location to simply be the mean of the points assigned to it, K-medoids will pick the point in the cluster that's closest to the mean as the new centroid, ensuring that the centroid is always assigned to a point that's actually in the dataset.

A completely different approach to simply computing distances is to build up a chain of datapoints that will become the cluster. DBSCAN takes two hyperparameters: epsilon (the maximum distance between points in a chain) and neighborhood size (the number of adjacent points necessary for the point to be marked an internal point). DBSCAN first picks a point in the dataset. Then, if the point is not an internal point, marks the point as noise. Else, if it is an internal point (has at least neighbor size neighbors), it begins a cluster by adding its neighbors to the list of unprocessed points. For each unprocessed point in the chain, DBSCAN marks it as in the cluster, then decides if it's an internal point or not. If it's an internal point, all of that point's unvisited neighbors are added to the list of unprocessed points. This continues until all unprocssed points are processed. Then, a new unclassified point is picked and the process repeats.

Lastly, a probabilistic model, the Gaussian Mixture Model, has several advantages over the others. First, it can say the probability that a point is in a cluster rather than just 0 or 1 for whether it's in or out. Second, it can generate new points in the cluster (as seen in Q4 above). GMM uses a process known as expectation maximization. Similar to how K-means picked initial centroids randomly, we pick initial probability distributions of each point for each cluster. So, for example, if we want three clusters, point 1's probability distribution might start at [0.2, 0.4, 0.4]. Then, now that we have baseline values for each point's cluster, we perform expectation maximization with those baseline values. After expectation maximization, we can recompute the probability that each point is in each cluster using the newly found theta coefficients, then those probabilities become the new baseline probability values. We repeat this process until convergence.

7. (15 points) Answer the following questions:

  1. How can you use association rules for classification?
  2. What can be one of the advantages to use the association rules for classification?
  3. Do you know some other model that can produce rules for the classification? Explain what is, and what should be the procedure to extract the rules.
  4. Explain why bias is needed to learn something.

1. Association rules basically tell how well a set is associated with another set. Put another way, given a set A, how likely is the set B to also show up in that context. This can provide a basis for classification. If A is associated with B, and B is associated with A, they should likely be classed together. For example, if diapers are associated with beer and beer is associated with diapers, they should both be classed together as "new parent products."

2. An item could belong to more than one class. For example, beer could be a "new parent product," but if it's also bought with hamburger patties and buns, you could also classify it as a "party product."

3. Classification trees lend themselves nicely to a rules-based approach, as each branch is essentially a rule. Starting at the top of the tree, we check a node's feature, then the boundary for the left/right divide. This is the condition for the root rule. You can think of it as the outermost if-else statement in a series of nested if-else statements. Then, you can follow the children and repeat the process, nesting the child's if-else in the corresponding if-else clause of the parent. When you reach a leaf node, just return the class.

4. The bias is exactly the thing that allows us to make predictions of data-points that we have not seen before. If our model didn't have bias, it would not be able to determine which class a data-point is if that data-point wasn't in the training data. Recalling one of the earlier lessons, if multiple hypotheses fit a data-point, without bias, there is no way to know which hypothesis to pick.
